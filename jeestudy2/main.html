Java Specification Requests (JSRs)

Some theory

The fundamental unit in Java EE is the Java Specification Request (JSR).

For example, Servlet 3.1 in Java EE 7 is specified in JSR 340, and Servlet 4.0 in Java EE 8 is specified in JSR 4.0.

A very informative document is Wikipedia article Java EE version history.

Some practice

Let's use the Java EE version history article to open the Servlet 4.0 specification.

The steps involve finding the the link, and opening the PDF.

Depending on JSRs, some other documents, such as a javadoc archive for the case of Servlet, may be available.

Java EE specifications and profiles

Some theory

Java EE 6 is specified in JSR 316, Java EE 7 is specified in JSR 342 and Java EE 8 is specified in JSR 366.

Each of these documents specifies which other JSRs are to be included, and how they must interact with each other.

All Java EE versions define two profiles.

The full profile includes everything.

The web profile includes only a subset of the Java EE specifications.

Let's open the Java EE 8 full profile specification and check that it includes the Servlet 4.0 specification

Java EE specification and Maven dependencies

Some theory

Software created for Java EE is supposed to be application server agnostic.

But some classes must still be injected in the IDE classpath for autocompletion purposes.

To solve this issue, Maven dependcies can be annotated with the provided scope, so that the corresponding JARs are available during code authoring and compilation, but not included in the final artifact.

Maven dependencies for Java EE are available in Maven central for these purposes, under the artifact ids javaee-api or javaee-web-api.

Other JSRs are references as dependencies, and can be included individually.

Some practice

A good place to search for maven dependencies is mvnrepository.com.

Let's suppose we are targetting Java EE 8 full profile. Then we need to search for javaee-api, from which we select version 8.0.

If we were targetting the Java EE 7 web profile, we would search for javaee-web-api and select version 7.0.

Let's suppose we want to include only the Servlet JSR for Java EE 7. Then we can inspect the compile depencies of >javaee-api to select just that.

Java EE application servers

Some theory

GlassFish is the reference Java EE implementation. It supports both the web profile and the full profile.

Other full profile implementations include Oracle WebLogic and IBM WebSphere.

Some other application servers, such as Apache Tomcat only include a subset of the Java EE JSRs. In these case, it is necessary to include the corresponding Maven dependencies manually.
